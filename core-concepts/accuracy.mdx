Path publishes one accuracy number for the predictions it acts on.
The same number appears on every Path surface (landing page, partner
exports, this page). It is recomputed every 30 minutes from canonical
SQL and pinned by an integrity verifier.

## The number

<Card title="Live 1-day directional accuracy" icon="bullseye">
  **78%** on the predictions Path acts on, over the last 14 days
  (n = 808 high-conviction calls), vs. the DeFiLlama market baseline
  of **53.5%**.
</Card>

## What "the predictions Path acts on" means

Path emits a directional call for every pool, every 15 minutes. Most
of those calls are low-confidence (the model is genuinely unsure) and
Path does not act on them. The 78% number is computed on the subset
the engine flags as high-conviction. That subset is roughly 5% of all
emissions and is the operational range a strategy manager would
actually trade.

The 53.5% baseline is the directional accuracy you would get by
trusting the DeFiLlama random-walk baseline on the same pools over
the same window. The lift is statistically significant at p < 0.001.

## Integrity layer

A continuous-loop verifier recomputes every accuracy number shown on
any Path surface (admin dashboards, this site, partner-facing exports)
from canonical SQL every 30 minutes. Drift greater than 0.5
percentage points pages on-call and auto-files a ticket. Every
number on this site is the number the verifier last published.

## Honest framing

Path does not publish a single test-set accuracy number, because
test-set accuracy is not what a partner integrates against. The
number above is live, forward-resolved, and reproducible from the
SQL the verifier runs.
