Signal Mesh is delivered to partners through two embedding surfaces
and an on-chain feed. Endpoint detail, schema, and access tokens are
handled inside the integration conversation; this page describes the
capability shape so a partner can decide whether the fit is right.

## Private API

Path operates a private API that publishes the prediction stream, the
liquidity-impact context, the historical accuracy registry, and the
rebalance-cost estimator. A partner can drive their own allocation UI
from this stream, run backtests over historical Path output, or wire
it into an existing portfolio system.

Access is granted per integration; pricing, rate-limits, and
data-residency posture are set in the partnership contract. To
request access, contact [hello@pathprotocol.finance](mailto:hello@pathprotocol.finance).

## Embeddable widget

A drop-in widget renders Path's allocation intelligence inside the
partner's earn or treasury surface. The widget reads the connected
wallet (or an explicit balance supplied by the partner), surfaces the
allocation breakdown, the recommended rebalance with confidence and
break-even, the liquidity-impact context for any deposit size, and
an expandable panel showing the signal and its accuracy band.

The widget is design-system aware: the partner provides colour
tokens, typography tokens, and a layout container, and the widget
styles itself to match.

## On-chain signed signals

For partners whose risk posture requires on-chain verifiability,
Path publishes the prediction stream through a Chainlink Runtime
Environment workflow. The workflow runs in a decentralised oracle
network, signs the prediction tuple, and posts the result on-chain.
Any vault contract on any supported chain can read the signed signal
with cryptographic guarantees. The on-chain feed is the recommended
shape for partners who want verifiability instead of operational
trust in Path's off-chain compute.
