Security is the precondition for institutional integration. Path is
non-custodial intelligence: the trust boundary is narrow by design, and
this page documents where that boundary sits.

## Custody model

Path never holds user funds. Capital deposited into a partner rail
stays with that rail's custodian or, where the rail allocates directly,
inside the underlying yield protocol's smart contracts. Path is read-
only against on-chain state and produces signals; the partner's
execution stack is what moves capital.

<CardGroup cols={2}>
  <Card title="What Path holds" icon="circle-xmark">
    Nothing. No custody, no withdrawal authority, no transaction signing
    on behalf of users.
  </Card>
  <Card title="What the partner holds" icon="vault">
    Full custody, full execution authority, full discretion over which
    signals to act on and which to defer.
  </Card>
</CardGroup>

The trust boundary is: Path produces signals, the partner executes.
Anything that crosses that boundary is a partner decision.

## Signed signal output

For partners that need on-chain verifiability rather than off-chain
compute trust, Path is wiring signal output through a Chainlink CRE
workflow. The DON-signed feed lets the consuming contract verify that
the signal it is reading was produced by Path's pipeline and was not
tampered with in transit.

<Card title="Chainlink CRE Early Access" icon="link" href="/integrations/integration-shape">
  Verifiable signal delivery for partners whose execution logic runs
  on-chain.
</Card>

Off-chain consumers can continue to use the API; the signed feed is an
add-on for partners that need stronger delivery guarantees.

## Operational security

Path operates two surfaces of its own: the public API and the Signal
Mesh prediction pipeline.

<Steps>
  <Step title="Deploy isolation">
    Production deploys run on Railway in an isolated project. Staging
    and production are separated at the infrastructure level. Secrets
    are managed through the platform's secret store and never live in
    source control.
  </Step>
  <Step title="Pipeline cadence">
    The Signal Mesh pipeline runs every 15 minutes against canonical
    on-chain and market data sources. Pipeline failures page on-call
    automatically.
  </Step>
  <Step title="Continuous verification">
    A verifier daemon re-computes every number published on Path
    surfaces from canonical SQL every 30 minutes. Drift beyond the
    published tolerance auto-files an incident ticket and pages
    on-call.
  </Step>
  <Step title="Access controls">
    Database access is role-isolated. Read access is logged. Write
    access to production data is restricted to the pipeline service
    account and a small set of operator identities.
  </Step>
</Steps>

## Audit roadmap

Path is pre-launch. We have not commissioned a third-party smart-
contract audit because there is no smart contract surface yet. The
audit roadmap follows the on-chain surface as it lands.
