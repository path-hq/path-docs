Security is the precondition for institutional integration. Path is
non-custodial intelligence: the trust boundary is narrow by design, and
this page documents where that boundary sits.

## Custody model

Path never holds user funds. Capital deposited into a partner rail
stays with that rail's custodian or, where the rail allocates directly,
inside the underlying yield protocol's smart contracts. Path is read-
only against on-chain state and produces signals; the partner's
execution stack is what moves capital.

<CardGroup cols={2}>
  <Card title="What Path holds" icon="circle-xmark">
    Nothing. No custody, no withdrawal authority, no transaction signing
    on behalf of users.
  </Card>
  <Card title="What the partner holds" icon="vault">
    Full custody, full execution authority, full discretion over which
    signals to act on and which to defer.
  </Card>
</CardGroup>

The trust boundary is: Path produces signals, the partner executes.
Anything that crosses that boundary is a partner decision.

## Operational security

- **Data isolation.** Production and staging are separated at the
  infrastructure level. Secrets live in a managed secret store, never
  in source control.
- **Continuous verification.** Every number published on a Path
  surface is re-computed from canonical SQL on a regular cadence;
  drift beyond the published tolerance auto-files an incident and
  pages on-call.
- **Access controls.** Database access is role-isolated. Read access
  is logged. Write access to production data is restricted to the
  pipeline service account and a small set of operator identities.
